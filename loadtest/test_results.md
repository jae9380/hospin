### Layer 섹션 (Hexagonal 레이어 분석)

| 패널명                     | 확인 메트릭                     |  정상   |    경고     |  위험   | 기록 항목       |
| -------------------------- | ------------------------------- | :-----: | :---------: | :-----: | --------------- |
| Presentation p95 응답 시간 | API 전체 응답 시간              | < 500ms |  500ms~1s   |  > 1s   | 엔드포인트별    |
| Application 처리 지연      | Presentation - Application 차이 | < 50ms  | 50ms~200ms  | > 200ms | 레이어 오버헤드 |
| Infrastructure 응답 시간   | DB/외부 의존성 응답 시간        | < 100ms | 100ms~300ms | > 300ms | 쿼리 성능 기준  |

**레이어 비교 해석 가이드:**

```
[케이스 1] Infrastructure >> Application
  Infrastructure: 500ms, Application: 50ms
  → DB 쿼리가 병목. 인덱스 추가 또는 쿼리 최적화 필요

[케이스 2] Application >> Infrastructure
  Infrastructure: 50ms, Application: 500ms
  → Java 비즈니스 로직이 병목. 알고리즘 개선 필요
  → GET /api/hospital/nearby에서 이 패턴 예상 (Haversine 계산)

[케이스 3] Presentation >> (Application + Infrastructure)의 합
  → AOP 오버헤드, JSON 직렬화, Security Filter 체인 등 확인
```
